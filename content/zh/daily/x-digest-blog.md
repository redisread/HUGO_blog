### 1️⃣ AI Agent 架构方法论

**原文作者：** @IntuitMachine  
**核心观点：** Agent 失败不是因为模型不够聪明，而是因为围绕模型的循环机制出了问题。

**深度解读：**

这篇推文提出了 Agent 开发的"循环优先"方法论，颠覆了常见的"模型优先"思维：

**关键洞察：**
- **真实任务 > 任务描述**：大多数 Agent 运行在摘要和陈旧指令上，应该让它处理正在发生的真实事情
- **缺失的循环**：几乎所有失败的 Agent 都缺少相同的组件：步骤间无记忆、无输出检查、无实际发生记录
- **验证优先于信任**：在不可逆操作（发送、删除、发布、支付）之前必须设置检查门
- **状态外部化**：如果 Agent 的记忆只存在于上下文窗口，下次压缩时就会丢失

**可实践建议：**
1. 从原始 trace 调试，而非 pass/fail 分数
2. 链式连接 Agent，而非让规划者向专家发号施令
3. 让每个 Agent 将前一个 Agent 的完成工作作为原始输入
4. 构建自我改进的循环：每次失败都成为信号，将 trace 反馈回去
